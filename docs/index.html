Menu Bar Usage Monitor

Know every reset before it lands.

TokenBar keeps session + weekly limits (and credits) visible in your menu bar, so you can pace output and ship with fewer surprises.

macOS 14+ Apple Silicon + Intel Free & open source No passwords stored
Download Latest View on GitHub

Requirements: macOS 14+ (Apple Silicon + Intel).

Homebrew: brew install --cask tokenbar/tap/tokenbar

Linuxbrew (CLI only): brew install tokenbar/tap/tokenbar

GitHub Releases builds update via Sparkle. Homebrew installs update via brew upgrade.

  • One status item per provider (Codex, Claude, Cursor, Gemini, Antigravity, Droid, Copilot, z.ai).
  • Session + weekly bars with reset times and optional credits.
  • Web-backed limits via browser cookies (Codex, Claude, Cursor, Droid).
  • Two-bar icon: session on top, weekly on bottom, dimmed on errors.
  • Minimal UI with no Dock icon.

Install + Updates

Use GitHub Releases or Homebrew. Sparkle auto-updates are enabled on GitHub builds only.

Providers

Codex · Claude Code · Cursor · Gemini · Antigravity · Droid · Copilot · z.ai. Open to new providers via the provider guide.

Privacy

Uses existing browser cookies for dashboard access. No passwords stored. Details in issue #12.

Usage Tracking

Session + weekly meters with reset countdowns, plus optional credits and code review limits.

Status + Widgets

Provider status polling with incident badges and WidgetKit snapshots that mirror the menu card.

CLI + Automation

Bundled tokenbar CLI for scripts and CI, with Linux CLI builds on GitHub Releases.

macOS Permissions

Full Disk Access only for Safari cookies, Keychain for tokens, and Files/Folders prompts as needed.
